## 二、Context / Hooks

### 2.1 I18nContext — 全局多语言上下文

所有页面组件可通过 `useI18n()` 获取当前语言状态和翻译函数。

```tsx
import { useI18n } from "../../lib/i18n";

function MyComponent() {
  const { t, lang, setLang } = useI18n();
  
  // t("key") → 返回当前语言的字符串
  // lang    → "zh" | "fr"
  // setLang → (lang: "zh" | "fr") => void
}
```

**Context 提供值：**

| 属性 | 类型 | 说明 |
|------|------|------|
| lang | `"zh" \| "fr"` | 当前语言 |
| setLang | `(lang: Lang) => void` | 切换语言 |
| t | `(key: TranslationKeys) => string` | 翻译函数，key 必须是翻译字典中的键 |

### 2.2 Provider 使用方式

```tsx
// App.tsx 顶层包裹
import { I18nContext, translations } from "./lib/i18n";

<I18nContext.Provider value={{ lang, setLang, t }}>
  <App />
</I18nContext.Provider>
```

## 三、类型定义

### 3.1 Lang / TranslationKeys

```tsx
// src/lib/i18n.ts
type Lang = "zh" | "fr";
type TranslationKeys = keyof typeof translations.zh;
// 即所有 zh 对象的 key: "navFeatures" | "heroTitle1" | "loginTitle" | ...
```

### 3.2 FeatureItem / ScreenshotPlaceholder / UploadFlowStatus

```tsx
// src/features/home/type.ts
interface FeatureItem {
  title: string;
  description: string;
  svg: React.ReactNode;
}

interface ScreenshotPlaceholder {
  label: string;
  desc: string;
  gradient: string;
}

type UploadFlowStatus = "idle" | "login-prompt" | "uploading" | "success";
```
