### 2.4 framer-motion Mock 说明

所有测试文件统一 mock framer-motion：

```tsx
vi.mock("framer-motion", () => {
  const createMotionComponent = (tag: string) => {
    const Component = (props: Record<string, unknown>) => {
      const { children, ...rest } = props;
      const {
        initial: _i, animate: _a, exit: _e, transition: _t,
        ...cleanProps
      } = rest as Record<string, unknown>;
      return React.createElement(tag, cleanProps, children as React.ReactNode);
    };
    Component.displayName = `motion.${tag}`;
    return Component;
  };

  return {
    motion: new Proxy(
      {},
      { get: (_target, tag: string) => createMotionComponent(tag) },
    ),
    AnimatePresence: ({ children }: { children: React.ReactNode }) =>
      React.createElement(React.Fragment, null, children),
  };
});
```

SplashScreen 和 DocumentViewer 的 mock 额外过滤了 `whileInView`, `whileHover`, `whileTap`, `viewport`, `variants`, `layout`, `layoutId` 等 framer-motion 独有 prop，防止它们被透传到原生 DOM 元素。

### 2.5 计时器测试模式

SplashScreen 和 App 测试使用 `vi.useFakeTimers()` / `vi.useRealTimers()` 控制 setTimeout 行为：

```tsx
import { act } from "@testing-library/react";

beforeEach(() => { vi.useFakeTimers(); });
afterEach(() => { vi.useRealTimers(); cleanup(); });

// 推进时间到指定毫秒数
act(() => { vi.advanceTimersByTime(800); });   // 触发 phase 2
act(() => { vi.advanceTimersByTime(1700); });  // 触发 phase 3
act(() => { vi.advanceTimersByTime(3800); });  // 触发 auto-dismiss
```

## 三、常见问题排查

### 3.1 测试失败

**问题：** `Objects are not valid as a React child (found: [object HTMLDivElement])`

**原因：** framer-motion mock 返回了原生 DOM 元素而非 React 元素。

**解决方案：** 检查测试文件中 framer-motion mock 的 `createMotionComponent` 函数，确保使用 `React.createElement` 而非 `document.createElement`。

```tsx
// 正确写法
return React.createElement(tag, cleanProps, children as React.ReactNode);

// 错误写法（会导致上述错误）
const el = document.createElement(tag);
el.textContent = children;
return el;
```

### 3.2 SplashScreen 动画不显示

**排查步骤：**
1. 检查 `SplashScreen.tsx` 中 `phase` 状态是否正确切换
2. 打开浏览器 DevTools，确认 `motion.div` 的 style 包含正确的 opacity/transform
3. 确认 `AnimatePresence mode="wait"` 包裹了条件渲染

### 3.3 左侧面板不显示

**排查步骤：**
1. 检查 `WorkspacePage` 中的 `leftMode` 状态（`null` 时不显示面板）
2. 点击左侧工具栏的「文件管理」按钮 → 应切换 leftMode 为 `"files"`
3. 确认 `showLeftPanel = leftMode !== null` 条件为 true
4. 检查 `motion.aside` 的 `animate` prop 中 `width` 和 `opacity` 是否正确

### 3.4 文档内容不显示

**排查步骤：**
1. 确认已通过文件管理器选中一个文件（左侧面板切换到章节树模式）
2. 确认已点击章节树中的某个章节
3. 检查 `DocumentViewer` 的 `filteredBlocks` — 需要 `selectedSectionId` 与 `ContentBlock.sectionId` 匹配
4. 检查模拟数据的 `contentBlocks` 中，各块的 `sectionId` 字段是否与章节 ID 对应

### 3.5 右侧面板拖拽不流畅或行为异常

**排查步骤：**
1. 检查 `WorkspacePage` 中 `setRightW` 回调的 delta 计算
2. 确认 `rightDragRef.current` 在 mousedown 时正确设置
3. 检查 mouseup 事件的自动折叠逻辑：面板宽度 < 189px 且光标距窗口右边缘 < 50px 时折叠
4. 确认 resize 事件的自动折叠条件

### 3.6 聊天消息内容为空

**排查步骤：**
1. 检查 `workspaceReducer` 中 `SEND_CHAT_MESSAGE` action 的 `content` 构建逻辑
2. 确认 `state.chatInput` 或 `state.quotedText` 非空
3. 确认负载传递正确（`message: {} as any` 是模拟占位）

3. 在 `DocumentToolbar` 中添加对应的工具栏按钮
4. 在 `DocumentViewer.test.tsx` 中添加格式测试

### 4.4 接入真实 AI 聊天

```tsx
// 在 WorkspacePage.tsx 中修改 SEND_CHAT_MESSAGE 处理
// 当前是模拟回复，替换为真实的 API 调用

case "SEND_CHAT_MESSAGE": {
  const userMsg = { ... };
  // 发起异步 API 请求
  fetch("/api/chat", {
    method: "POST",
    body: JSON.stringify({ message: state.chatInput, quotedText: state.quotedText }),
  }).then(res => res.json()).then(data => {
    dispatch({ type: "ADD_ASSISTANT_MESSAGE", message: data.reply });
  });
  return { ...state, chatMessages: [...state.chatMessages, userMsg], chatInput: "", loading: true };
}

case "ADD_ASSISTANT_MESSAGE":
  return { ...state, chatMessages: [...state.chatMessages, action.message], loading: false };
```

### 4.5 添加新的面板布局变化

工作区支持左/右两侧面板，布局代码集中在 `WorkspacePage.tsx`：

1. 修改 `leftMode` 类型（当前为 `"files" | "sections" | null`）以支持新面板
2. 在左侧工具栏中添加新的切换按钮
3. 在左侧面板区域添加新的 `motion.div` 和 `AnimatePresence` 切换逻辑
4. 调整宽度约束（如果有新的最小/最大宽度要求）

### 4.6 接入真实文件上传

// 当前 FileManager 的 handleUpload 仅创建本地模拟文件

const handleUpload = async (e: React.ChangeEvent<HTMLInputElement>) => {
  const file = e.target.files?.[0];
  if (file) {

    formData.append("file", file);
    try {
      const res = await fetch("/api/upload", { method: "POST", body: formData });
      const data = await res.json();

当前项目无需环境变量。后续接入真实后端时需添加：

```
VITE_API_BASE_URL=http://localhost:8000/api
```

### 7.2 构建产出

```
npm run build
# 产出在 dist/ 目录
# Tauri 模式下前端产物会被 Tauri 打包进桌面应用
```

### 7.3 浏览器兼容性

- 使用 `contentEditable` 实现富文本编辑，兼容现代浏览器
- `navigator.share` 仅在支持 Web Share API 的浏览器（移动端/最新 Chrome）下可用
- 不支持 IE 等老旧浏览器
