# 首页模块 — 维护手册

> 本文档说明首页模块的日常维护、常见问题排查、测试运行和扩展指南，供运维和后续开发者使用。

---

## 一、本地开发

### 1.1 启动开发服务器

```bash
# 前端开发服务器（端口 5173）
npm run dev

# 或 Tauri 桌面开发模式
npm run tauri -- dev
```

### 1.2 运行测试

```bash
# 运行所有前端测试（含覆盖率）
npx vitest run --coverage

# 仅运行首页模块相关测试
npx vitest run src/features/home/
npx vitest run src/components/ui/Navbar.test.tsx

# 查看覆盖率报告
open coverage/frontend/index.html    # Windows: start coverage/frontend/index.html
```

### 1.3 构建

```bash
npm run build
# 产出在 dist/ 目录
```

### 1.4 代码检查

```bash
npx tsc --noEmit           # 类型检查
npx eslint src/features/home/   # 代码风格（如果配置了 ESLint）
```

## 三、常见问题排查

### 3.1 测试失败

**问题：** `Objects are not valid as a React child (found: [object HTMLDivElement])`

**原因：** framer-motion mock 返回了原生 DOM 元素而非 React 元素。

**解决方案：** 检查 `HomePage.test.tsx` 中 framer-motion mock 的 `createMotionComponent` 函数，确保使用 `React.createElement` 而非 `document.createElement`。

```tsx
// 正确写法
return React.createElement(tag, cleanProps, children as React.ReactNode);

// 错误写法（会导致上述错误）
const el = document.createElement(tag);
el.textContent = children;
return el;
```

### 3.2 翻译未生效

**问题：** 切换语言后页面文案未更新。

**排查步骤：**
1. 检查组件是否使用了 `useI18n()` hook
2. 检查翻译 key 是否在 `translations.zh` 和 `translations.fr` 中都存在
3. 运行 `npx vitest run src/lib/i18n.test.ts` 确认测试通过
4. 确认该 key 对应的值在两个语言中都不为空

### 3.3 暗黑模式不生效

**排查步骤：**
1. 确认 `App.tsx` 中 `div` 的 `className` 是否正确包含 `dark`
2. 确认样式使用 `dark:` 前缀（如 `dark:bg-black`）
3. 打开浏览器 DevTools，检查 `<html>` 或外层 `<div>` 是否有 `dark` class
4. 检查 `tailwind.config.js` 中 `darkMode` 配置（v4 使用 `@media` 策略，class 通过 `div.dark` 控制）

### 3.4 横向滑动不流畅

**问题：** 功能卡片区域横向滑动卡顿或对齐不正确。

**解决方案：**
- 确保固定了 `min-w-[280px]` 宽度
- 使用 `snap-x snap-mandatory` 配合 `snap-start`
- 使用 `overflow-x-auto` 而非 `overflow-x-scroll`
- 已内置 `scrollbar-hide` 类隐藏滚动条

## 四、扩展指南

### 4.1 添加新翻译 key

1. 在 `src/lib/i18n.ts` 的 `translations.zh` 和 `translations.fr` 对象中同时添加同一个 key
2. 在需要使用翻译的组件中：
   ```tsx
   const { t } = useI18n();
   // 使用
   <span>{t("yourNewKey")}</span>
   ```
3. 在 `i18n.test.ts` 中添加测试验证新 key 的值

### 4.2 添加新的 SVG 图标

1. 在 `src/features/home/constants.tsx` 的 `SVG_ICONS` 对象中添加：
   ```tsx
   export const SVG_ICONS = {
     // ... 已有图标
     yourIcon: (
       <svg width="24" height="24" viewBox="0 0 24 24" fill="none" stroke="currentColor" strokeWidth="1.5" ...>
         <path d="..." />
       </svg>
     ),
   } as const;
   ```
2. 使用时 `import { SVG_ICONS } from "../constants"`
3. 在 `constants.tsx` 的测试文件中验证新图标

### 4.3 添加新的功能特性卡片

```tsx
// 在 HomePage.tsx 的 features 数组中添加
const features = [
  // ... 已有特性
  {
    title: t("feature5Title"),     // 先在 i18n.ts 添加翻译
    description: t("feature5Desc"),
    svg: SVG_ICONS.yourNewIcon,    // 或在 constants.tsx 添加 SVG
  },
];
```

### 4.4 替换截图占位为真实截图

```tsx
// 在 HomePage.tsx 的 screenshots 数组中已有占位项
// 将截图区域的占位 div 替换为真实的 <img> 标签
<div className="aspect-[16/10] bg-gradient-to-br ${SCREENSHOT_GRADIENTS[idx]} flex items-center justify-center">
  {/* 替换为 */}
  <img src="/screenshots/course-overview.png" alt={item.label} className="w-full h-full object-cover" />
</div>
```

### 4.5 接入真实登录

1. 创建认证上下文（如 `src/lib/auth.ts`）：
   ```tsx
   const AuthContext = createContext({
     isLoggedIn: false,
     login: () => {},
     logout: () => {},
   });
   ```
2. 修改 `App.tsx` 中的 `openLogin` 逻辑：
   ```tsx
   const [showLogin, setShowLogin] = useState(false);
   const [isLoggedIn, setIsLoggedIn] = useState(false);

   const handleUploadClick = () => {
     if (!isLoggedIn) {
       setShowLogin(true);
     } else {
       // 打开文件选择器
     }
   };
   ```
3. LoginModal 的"继续登录"按钮调用真实的登录 API

## 五、CI/CD 配置

### 5.1 GitHub Actions

参考 `.github/workflows/quality.yml`，首页模块相关的 CI 步骤：

```yaml
# 前端测试
- run: npm ci
- run: npm run test:frontend           # 运行所有前端测试
- run: npm run test:frontend:coverage   # 生成覆盖率报告
```

### 5.2 SonarQube 配置

在 `sonar-project.properties` 中首页模块相关配置：

```properties
sonar.sources=src
sonar.tests=src
sonar.test.inclusions=**/*.test.ts,**/*.test.tsx
sonar.javascript.lcov.reportPaths=coverage/frontend/lcov.info
sonar.exclusions=src/components/ui/**
```

首页模块的测试代码在 `src/features/home/` 和 `src/components/ui/` 路径下，均在 SonarQube 扫描范围内（`src/components/ui/**` 仅排除 UI 组件源码，测试文件在 `src/` 下仍可被扫描）。
