## 二、数据流

### 2.1 登录流程

```
用户点击上传/登录按钮
        │
        ▼
UploadZone.onLogin() / Navbar.onLogin()
        │
        ▼
App.tsx openLogin() → setShowLogin(true)
        │
        ▼
LoginModal 显示（带 backdrop-blur 遮罩）
        │
        ├── 用户点击"继续登录" → 后续接入真实 auth
        │
        └── 用户点击"稍后再说"/遮罩 → onClose → setShowLogin(false)
```

### 2.2 多语言切换

```
用户点击 Navbar 语言按钮
        │
        ▼
Navbar → setLang(lang === "zh" ? "fr" : "zh")
        │
        ▼
I18nContext 更新 lang 值
        │
        ▼
所有使用 useI18n() 的组件重新渲染
HomePage / Navbar / LoginModal / UploadZone 文案切换
```

### 2.3 暗黑模式切换

```
用户点击 Navbar 暗黑按钮
        │
        ▼
Navbar → setDark(prev => !prev)
        │
        ▼
App.tsx 更新 dark 状态
        │
        ▼
div 的 className 在 "dark" / "" 之间切换
        │
        ▼
所有 Tailwind dark: 前缀的样式生效/失效
```

### 2.4 滚动定位

```
用户点击 Hero 区域"开始使用" / 底部"免费开始使用"
        │
        ▼
scrollToWorkflow()
        │
        ▼
workflowRef.current.scrollIntoView({ behavior: "smooth", block: "start" })
        │
        ▼
页面滚动到"三步搞定课程解析"标题顶部
```

## 三、关键实现细节

### 3.1 横向滑动卡片

```tsx
// Features 和 Screenshots 区域使用 CSS snap-scroll
<div className="flex gap-4 overflow-x-auto snap-x snap-mandatory scrollbar-hide">
  {items.map(item => (
    <div className="min-w-[280px] flex-shrink-0 snap-start">
      {/* 卡片内容 */}
    </div>
  ))}
</div>
```

- `scrollbar-hide` 全局 CSS 类（在 `index.css` 定义）隐藏滚动条
- `snap-x snap-mandatory` 确保每次滚动对齐到卡片起始位置
- Features 区域额外有左右箭头按钮调用 `scrollRef.current.scrollBy()` 滚动

### 3.2 拖拽上传

```tsx
// UploadZone 使用 HTML5 Drag & Drop API
onDragOver → e.preventDefault() + setDragging(true)
onDragLeave → e.preventDefault() + setDragging(false)
onDrop → 获取 file.name + 调用 onLogin() 弹出登录提醒
```

- 拖拽过程中区域会高亮（border + scale 动画）
- 支持的文件类型在 `SUPPORTED_FORMATS` 中配置
- 隐藏的 `<input type="file">` 用于点击选择文件
- 无论拖拽还是点击，都先触发登录再处理文件

### 3.3 登录浮框动画

```tsx
// LoginModal 使用 framer-motion AnimatePresence
<AnimatePresence>
  {open && (
    <motion.div
      initial={{ opacity: 0, scale: 0.95, y: 10 }}
      animate={{ opacity: 1, scale: 1, y: 0 }}
      exit={{ opacity: 0, scale: 0.95, y: 10 }}
      transition={{ duration: 0.25, ease: [0.16, 1, 0.3, 1] }}
    >
      {/* 浮框内容 */}
    </motion.div>
  )}
</AnimatePresence>
```

- `ease: [0.16, 1, 0.3, 1]` 是 Apple 风格的 spring easing
- backdrop-blur 遮罩层在浮框下方

### 3.4 i18n 系统

```tsx
// src/lib/i18n.ts
const I18nContext = createContext<I18nContextValue>({
  lang: "zh",
  setLang: () => {},
  t: (key) => translations.zh[key],
});

export function useI18n() {
  return useContext(I18nContext);
}
```

- Context 在 `App.tsx` 顶层提供，全局可用
- `t(key)` 类型安全（`TranslationKeys` 是所有 key 的联合类型）
- 翻译字典中 zh 和 fr 的 key 必须完全一致（有测试保障）

## 四、样式方案

### 4.1 Tailwind CSS v4

- 使用 `@import "tailwindcss"` 导入（v4 新语法）
- 暗黑模式通过 `dark:` 前缀 + 外层 `div.dark` class 控制
- 颜色系统：`neutral-500` / `neutral-400` 为灰色文字，`black/5` 为半透明黑色

### 4.2 Apple 风格设计

| 元素 | 实现 |
|------|------|
| 字体 | `-apple-system, SF Pro Display, SF Pro Text` |
| 圆角 | `rounded-full`（按钮），`rounded-3xl`（卡片） |
| 毛玻璃 | `backdrop-blur-xl bg-white/80`（Navbar） |
| 大标题 | `text-[2.6rem]` → `text-[4rem]`（响应式） |
| 渐变文字 | `text-transparent bg-clip-text bg-gradient-to-r` |
| 柔和阴影 | `shadow-lg shadow-black/10 dark:shadow-white/10` |

## 五、SEO 与可访问性

- 所有交互元素使用 `<button>` 标签（自带键盘可访问性）
- 导航链接使用 `<a href="#section-id">` 实现锚点跳转
- SVG 图标行内嵌入（无需外部资源加载）
- Footer 版权信息使用 `&copy;` HTML 实体
