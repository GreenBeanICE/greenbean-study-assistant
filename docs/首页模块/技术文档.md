# 首页与工作区模块 — 技术文档

> 本文档介绍首页启动屏及工作区模块的架构设计、组件树结构、数据流和关键实现细节，供开发者理解系统运作方式。

## 一、架构概览

### 1.1 应用启动流程

```
App.tsx
│
├── showSplash === true
│   └── SplashScreen (开场动画，3.8s 自动跳过)
│         ├── 相位 1: 占位圆点动画 (0-800ms)
│         ├── 相位 2: 显示"欢迎"文字 (800-1700ms)
│         └── 相位 3: 显示副标题 (1700-3800ms)

```

## 二、数据流

### 2.1 页面切换流程 (Splash → Workspace)

```
App 首次渲染
      │
      ▼
showSplash = true → 显示 SplashScreen
      │
      ├── 自动流程: 3.8s 后自动调用 onSkip
      │
      ├── 用户操作: 点击"跳过"按钮 → onSkip
      │
      ▼
setShowSplash(false) → SplashScreen exit 动画 (AnimatePresence)
      │
      ▼
AnimatePresence 切换到 WorkspacePage
      │
      ▼
WorkspacePage 创建初始 state + 渲染三栏布局
```

### 2.2 文件选择 → 章节导航流程

```
用户点击文件名 (如 "cours-analyse-s1.pdf")
      │
      ▼
FileManager.handleFileClick → onFileSelectWithName(id, name)
      │
      ▼
WorkspacePage.handleFileSelect
      ├── setSelectedFileId(id)
      ├── setSelectedFileName(name)
      └── setLeftMode("sections")  → 切换到章节树面板
      │
      ▼
用户点击章节 (如 "1.1 背景介绍")
      │
      ▼
SectionTree.onSelect(id) → WorkspacePage.d(id)
      │
      ▼
dispatch(SELECT_SECTION, sectionId = "ch1-1")
      │
      ▼
DocumentViewer 过滤 contentBlocks → 显示对应章节内容
ScrollIntoView 滚动到对应 DOM 节点
```

### 2.3 聊天消息流程

```
用户在 ChatPanel 输入文字 → onInputChange → dispatch(SET_CHAT_INPUT)
      │
      ▼
用户按 Enter / 点击发送按钮 → onSend
      │
      ▼
WorkspacePage.send() 判断:
  ├── 有空输入且无引用 → 不发送
  └── 有输入或引用 → dispatch(SEND_CHAT_MESSAGE)
        │
        ▼
workspaceReducer.SEND_CHAT_MESSAGE:
  ├── 创建 user message (有引用时加上 "[引用] xxx")
  ├── 创建 assistant message (模拟回复)
  ├── 清空 chatInput 和 quotedText
  └── tokenUsage += 150
        │
        ▼
ChatPanel 重新渲染 → 新消息显示在消息列表底部
scrollIntoView 自动滚动到底部
```

### 2.4 文本选中 → 引用流程

```
用户在 DocumentViewer 选中文字
      │
      ▼
selectionchange 事件 → setHasSelection(true) → 工具栏启用
      │
      ▼
用户选择 "引用此段询问 AI"
      │
      ▼
SelectionMenu.onQuote → dispatch(QUOTE_SELECTION)
      │
      ▼
quotedText 被设置 → ChatPanel 显示 QuoteBar
      │
      ▼
用户在 ChatPanel 输入 + 发送 → 消息带引用前缀 "[引用] xxx"
```

### 2.5 面板折叠/展开流程

```
用户点击左侧工具栏按钮:
  ├── 文件管理按钮 → toggleLeftPanel("files")
  │     ├── leftMode 切换 (files ↔ null)
  │     └── leftCollapsed true 时先执行 TOGGLE_LEFT_PANEL
  │
  ├── 章节导航按钮 → toggleLeftPanel("sections")
  │     └── 同上（仅在文件选中时可用）
  │
  └── AI 聊天按钮 → toggleRightPanel
        └── dispatch(TOGGLE_RIGHT_PANEL)

自动折叠机制:
  ├── 鼠标拖拽右侧面板到最小阈值 (189px) 且光标在窗口右边缘 50px 内 → 自动折叠
  └── 窗口 resize 导致可用空间 < 工具栏 + 左侧 + 189 + 200 → 自动折叠右侧面板
```

## 三、关键实现细节

### 3.1 SplashScreen 三相位动画

```tsx
// 使用 setTimeout 分阶段切换 phase
const [phase, setPhase] = useState<"dots" | "welcome" | "subtitle">("dots");

useEffect(() => {
  const t1 = setTimeout(() => setPhase("welcome"), 800);
  const t2 = setTimeout(() => setPhase("subtitle"), 1700);
  return () => { clearTimeout(t1); clearTimeout(t2); };
}, []);
```

- 占位圆点使用 `motion.div` 实现脉动动画 (`scale` + `opacity` 循环)
- 文字通过 `AnimatePresence mode="wait"` 实现淡入效果
- 3 个渐变气泡和 3 个实心点交错排列，营造动态背景

### 3.2 workspaceReducer 状态管理

```tsx
// 使用 useReducer 管理复杂工作区状态
const [state, dispatch] = useReducer(workspaceReducer, initialState);

// 使用 useCallback 封装 dispatch 调用，避免子组件不必要的重渲染
const d = useCallback((s: string) => dispatch({ type: "SELECT_SECTION", sectionId: s }), []);
```

- reducer 支持 20 种 action 类型，覆盖章节导航、内容编辑、聊天、面板布局
- 宽度约束：左侧面板 160-302px，右侧面板 189-340px
- 异步回调通过 `as any` 类型断言简化（类型定义待完善）

### 3.3 可拖拽分隔条

```tsx
// ResizableHandle 使用 document-level mousemove/mouseup
// 避免鼠标拖拽过快移出元素区域导致事件丢失
useEffect(() => {
  document.addEventListener("mousemove", handleMouseMove);
  document.addEventListener("mouseup", handleMouseUp);
  return () => {
    document.removeEventListener("mousemove", handleMouseMove);
    document.removeEventListener("mouseup", handleMouseUp);
  };
}, [onResize, onDragStateChange]);
```

- 拖拽时设置 `document.body.style.cursor = "col-resize"` 和 `userSelect = "none"`
- 支持左侧 (left) 和右侧 (right) 两种位置
- 可选双击 toggle 功能

### 3.4 内容编辑系统

```tsx
// 使用 contentEditable 实现所见即所得编辑
<span
  contentEditable
  suppressContentEditableWarning
  onBlur={(e) => {
    const val = e.currentTarget.textContent ?? "";
    if (val !== text) onBlur(val);
  }}
/>
```

- 文本行：支持直接编辑文字内容
- 表格：支持编辑表头和单元格
- 图片：支持编辑图例文字
- 通过 `UPDATE_LINE_TEXT` action 持久化编辑结果
- 通过 `FORMAT_LINE` action 切换加粗/斜体/下划线等格式

### 3.5 文档查看器横向滚动

```tsx
// 内容区域在内容宽度超过容器时启用横向滚动
<div className="min-w-max" style={{ minWidth: "max-content" }}>
  <div className="max-w-3xl mx-auto">
    {/* 内容块 */}
  </div>
</div>
```

- `min-w-max` 确保内容宽度不压缩，保护宽表格和大图片
- 容器 `overflow-x-auto` 允许用户横向滚动查看完整内容
- `scrollbar-hide` 全局类隐藏滚动条

## 四、样式方案

### 4.1 Tailwind CSS

- 使用 `@import "tailwindcss"`（v4 语法）
- 主色调：透明/白色背景 + black/5 边框
- 交互反馈：hover:bg-black/10, active:bg-blue-500/20
- AI 相关元素使用 `from-blue-500 to-purple-600` 渐变

### 4.2 布局特色

| 元素 | 实现 |
|------|------|
| 三栏布局 | `flex-1 flex overflow-hidden` 嵌套 `flex-shrink-0` |
| 可调面板 | 使用 `animate={{ width }}` 调整 + `transition` 动画 |
| 左侧工具栏 | 固定 `w-12`，竖向排列 |
| 折叠/展开 | 通过 `width: 0` + `opacity: 0` 配合 `overflow-hidden` |

## 五、状态管理总结

```
SplashScreen:   本地 useState 管理三相位
WorkspacePage:  useReducer + 多个 useCallback 封装 dispatch
FileManager:    本地 useState (内部文件列表、搜索、右键菜单)
DocumentViewer:  接收 Props 无状态 + 本地 selectionchange 监听
ChatPanel:      接收 Props 无状态
ResizableHandle: useRef 追踪拖拽状态 + useEffect 绑定/解绑文档事件
```

所有组件数据流遵循 **单向数据流**：WorkspacePage 持有顶层 State，通过 Props 向下传递数据，通过回调函数向上报告用户操作。
